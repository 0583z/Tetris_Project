# 俄罗斯方块：好友邀请对战版

基于 Pygame 的俄罗斯方块游戏，支持**单人模式**与**双人网络联机对战**，包含完整的好友大厅系统、实时对战数据同步、音效反馈和动态 BGM。

---

## 功能特性

### 🕹️ 单人模式
- 经典俄罗斯方块玩法，7-bag 随机发块算法
- 消行计分（连消 Combo 加成）、等级递增、下落速度随等级加快
- 本地最高分记录

### 🌐 双人网络对战
- **好友大厅系统**：注册用户名 → 登录大厅 → 添加好友 → 在线实时对战
- **好友管理**：发送/接收/同意/拒绝好友申请，实时在线状态显示
- **对战邀请**：向在线好友发起对战请求，弹出式邀请通知
- **攻击机制**：消除 2/3/4 行分别向对手发送 1/2/4 行干扰垃圾行
- **实时同步**：双方游戏画面实时传输，对手棋盘可视化展示
- **胜负判定**：一方 Game Over 后自动判定胜负

### 🎵 音效与音乐
- 移动、旋转、硬降、消行、攻击、游戏结束等独立音效
- Combo 消行音效音量动态递增
- 正常/危险（堆叠过高）双状态 BGM 自动切换
- 支持中文系统字体渲染

## 快速开始

### 1. 启动服务端

在服务器/本机上运行：

```bash
python server.py
```

服务端默认监听 `0.0.0.0:5555`，启动后会显示：

```
🚀 超级对战与好友大厅服务器已启动，正在端口 5555 守候...
```

### 2. 启动客户端

```bash
python client.py
```

在登录界面输入：
- **用户名**：任意名称（首次登录自动注册）
- **IP 地址**：服务器 IP（本机测试用 `127.0.0.1`）

点击「进入游戏大厅」即可进入大厅。

### 3. 进入大厅后的操作

| 操作 | 说明 |
|------|------|
| 🕹️ 单人模式 | 直接开始单人游戏，无需联网 |
| ➕ 扩充朋友圈 | 输入好友用户名，发送好友申请 |
| 📥 好友申请 | 查看收到的申请，同意或拒绝 |
| ⚡ 邀请对战 | 向在线好友发起对战邀请 |

### 4. 操作键位（游戏中）

| 按键 | 功能 |
|------|------|
| ← → | 左右移动 |
| ↑ | 旋转方块 |
| ↓ | 加速下落 |
| ESC | 返回大厅 |

## 通信协议

客户端与服务端通过 TCP Socket 进行 JSON 行协议通信（每条消息以 `\n` 结尾）。

### 主要消息类型

| type | 方向 | 说明 |
|------|------|------|
| `login` | C→S | 用户登录/注册 |
| `login_resp` | S→C | 登录响应，携带好友列表和申请 |
| `add_friend` | C→S | 发送好友申请 |
| `new_request` | S→C | 实时推送好友申请通知 |
| `accept_friend` / `reject_friend` | C→S | 同意/拒绝好友申请 |
| `friend_added` | S→C | 好友关系建立确认 |
| `friend_status` | S→C | 好友上下线状态广播 |
| `invite_friend` | C→S | 发起对战邀请 |
| `invite_received` | S→C | 收到对战邀请 |
| `accept_invite` / `reject_invite` | C→S | 接受/拒绝对战邀请 |
| `match_start` | S→C | 对战房间建立，双方进入游戏 |
| `game_data` | C⇄S | 实时棋盘数据同步转发 |
| `attack` | C⇄S | 攻击垃圾行转发 |
| `toast` | S→C | 系统提示消息 |

## 计分规则

- 消除行数 `n`，得分 = `n² × 100 × Combo 倍率`
- 每消除 10 行，等级 +1
- 下落速度随等级递增：`max(100, 450 - level × 40)` 毫秒/格

## 开发说明

- 客户端使用 `pygame` 进行渲染和事件处理
- 服务端使用标准库 `socket` + `threading` 实现多客户端并发
- 账号数据存储在 `server_accounts.json`，无需数据库
- 方块形状定义在 `SHAPES` 列表中，可扩展
- 音效资源放在 `assets/` 目录下，缺失时静默跳过
- 兼容 PyInstaller 打包（通过 `resource_path()` 函数）
